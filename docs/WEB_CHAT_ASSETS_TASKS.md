## 执行规则

后续每次完成本文件中的一个任务或一个明确子任务后，必须同步更新：

- `docs/WEB_CHAT_PROGRESS.md`

更新要求：

- 追加一条新记录，不覆盖旧记录
- 必须写明完成时间
- 必须写明完成任务名称
- 必须写明任务文档位置
- 必须写明完成内容
- 必须写明下一步应执行的任务

执行顺序要求：

1. 先完成代码、文档或配置变更
2. 再执行必要验证
3. 最后回写 `docs/WEB_CHAT_PROGRESS.md`

这条规则对后续所有 A0-A7 任务都生效。

## A0：数据与目录基础

**范围：** 建立聊天资产落盘目录和数据库结构，为上传与清理能力打底。

### 任务清单

- [ ] 在 `$HOME/.ngent/assets` 下建立统一资产目录
- [ ] 新增迁移：`uploads`
- [ ] 新增迁移：`storage_usage`
- [ ] 可选新增迁移：`upload_events`
- [ ] 初始化 `storage_usage(scope='global', max_bytes=5368709120)`
- [ ] 在 storage 层新增资产相关读写 API
- [ ] 补充目录创建与路径清理工具函数
- [ ] 统一“聊天资产”统计口径，不把 DB / 日志算入 `5GB`

### 验收标准

- 服务启动后若资产目录不存在，会自动创建
- 新数据库初始化后存在 `uploads`、`storage_usage` 表
- `storage_usage.max_bytes` 默认值为 `5368709120`
- 存储层可成功写入并查询一条附件记录

## A1：上传接口

**范围：** 实现上传入口，支持文件上传和图片粘贴上传。

### 任务清单

- [ ] 新增 `POST /v1/uploads`
- [ ] 支持 `multipart/form-data`
- [ ] 支持一次上传多个文件
- [ ] 按服务端生成的真实文件名落盘
- [ ] 记录原始文件名、MIME、大小、SHA256、路径等元数据
- [ ] 服务端基于文件头重新探测 MIME
- [ ] 添加文件类型白名单
- [ ] 添加单文件大小限制
- [ ] 添加单次请求总大小限制
- [ ] 上传成功后更新 `storage_usage.used_bytes`

### 验收标准

- 上传一个文本文件返回 `200`
- 上传一个图片文件返回 `200`
- 上传多个文件返回多条 `uploads`
- 超过单文件大小限制返回 `413`
- 类型不支持返回 `400 INVALID_ARGUMENT`
- 上传成功后文件真实存在于资产目录中

- 上传失败时不会产生脏消息

### 任务清单

- [ ] 新增 `GET /v1/storage`
- [ ] 维护 `used_bytes`
- [ ] 服务启动时扫描并修正资产统计

## A6：自动清理

**范围：** 聊天资产超出 `5GB` 后，按“最旧优先”自动清理。

### 任务清单

- [ ] 实现写入前预检查
- [ ] 实现写入后复核
- [ ] 实现启动时超限清理
- [ ] 实现周期性清理任务
- [ ] 按 `created_at ASC` 删除最旧附件
- [ ] 删除缩略图与原图时保持一致性
- [ ] 更新 `uploads.status=deleted`
- [ ] 更新 `storage_usage.used_bytes`
- [ ] 记录 `quota_deleted` 日志或事件
- [ ] 无法再释放空间时返回 `QUOTA_EXCEEDED`

### 验收标准

- 当聊天资产超过 `5GB` 时，系统自动清理最旧资产
- 清理后 `usedBytes <= maxBytes`
- 被清理的附件再访问返回 `404`
- 清理不会误删数据库、日志、项目源码
- 若没有可清理资产，上传返回 `409 QUOTA_EXCEEDED`

## 推荐实现顺序

建议严格按这个顺序推进：

1. A0 数据与目录基础
2. A1 上传接口
3. A2 前端待发送附件
4. A3 Turn 绑定附件
5. A4 下载与缩略图
6. A5 配额统计
7. A6 自动清理
8. A7 收尾与测试

这样可以避免过早实现清理逻辑时，基础数据模型还没定稳。

## 测试建议

优先增加这些测试：

- 上传文本文件成功
- 上传图片成功并生成缩略图
- 粘贴图片上传成功
- 跨 `X-Client-ID` 访问附件返回 `404`
- 超过单文件限制返回 `413`
- 超过 `5GB` 后触发最旧附件清理
- 清理后 `usedBytes` 回落
- 清理后历史消息中的附件显示“已清理”状态
